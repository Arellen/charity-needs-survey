# 公益微信小程序 · 技术实现方案

> 适用场景：捐款渠道、受助人录入、项目管理 三大核心模块
> 阅读对象：开发者
> 标注说明：`[必做]` `[可选]` `[依赖机构确认]`

---

## 一、项目概述

打通"募款→项目→受助人"全链路的微信小程序，核心能力：

- 捐款人浏览项目并微信支付捐款
- 一线人员/合作机构现场录入受助人信息
- 项目全生命周期管理（立项→筹款→执行→结项）
- `[可选]` 场景化捐款入口（嵌入合作商家场景）
- `[可选]` 企业配捐机制
- `[可选]` 尊严支付（受助人象征性付费）
- `[可选]` 合作机构多租户管理

## 七、核心业务逻辑设计

### 7.1 捐款流程（含配捐）

```
用户发起捐款 POST /api/donations
  → 后端创建捐款订单(status=pending)
  → 调用微信支付统一下单API
  → 返回支付参数给前端 → 前端调 wx.requestPayment()
  → 用户完成支付
  → 微信支付回调 POST /api/donations/callback
    → 验签
    → 更新捐款状态为 paid
    → 更新项目 raised_amount
    → [可选] 触发配捐匹配 processMatching()
    → [可选] 检查是否可以生成反馈
    → 生成电子证书（异步队列）
    → 返回给微信 SUCCESS
```

### 7.2 配捐匹配逻辑 `[可选]`

```typescript
// 捐款支付成功后触发
async function processMatching(donation: Donation, project: Project) {
  // 项目级配捐开关检查
  if (!project.matchingEnabled) return;

  // 查找有效配捐池
  const pool = await db.matchingPools.findFirst({
    where: {
      OR: [
        { projectId: project.id },
        { projectId: null },  // 全局配捐池
      ],
      status: 'active',
      remainingAmount: { gt: 0 },
      startDate: { lte: new Date() },
      endDate: { gte: new Date() },
    },
    orderBy: { remainingAmount: 'desc' },  // 优先消耗余额多的
  });

  if (!pool) return;

  // 计算配捐金额
  let matchAmount = Math.floor(donation.amount * pool.matchRatio);
  if (pool.maxPerDonation > 0 && matchAmount > pool.maxPerDonation) {
    matchAmount = pool.maxPerDonation;
  }
  if (matchAmount > pool.remainingAmount) {
    matchAmount = pool.remainingAmount;
  }
  if (matchAmount <= 0) return;

  // 事务：原子扣减
  await db.transaction(async (tx) => {
    await tx.matchingPools.update({
      where: { id: pool.id },
      data: {
        usedAmount: { increment: matchAmount },
        status: pool.remainingAmount - matchAmount <= 0 ? 'exhausted' : undefined,
      },
    });

    await tx.donations.update({
      where: { id: donation.id },
      data: {
        matchedAmount: matchAmount,
        matchingPoolId: pool.id,
        totalImpact: donation.amount + matchAmount,
      },
    });
  });
}
```

### 7.3 捐款反馈闭环 `[可选]`

```
捐款完成 → 善款进入项目池 → 项目资金分配给受助人 → 帮扶执行
  → 系统将捐款与受助人帮扶记录关联（1:N匹配）
  → 生成反馈内容："您的捐款[金额]已用于帮助[地区][数量]名[困难类型]对象[帮扶内容]"
  → 通过订阅消息推送通知捐款人
```

**匹配策略**：不要求严格的 1:1（你的 100 元精确对应张三），而是按时间窗口聚合。比如：某用户的捐款在本月被分配到该项目本月帮扶的受助人群体中。

```typescript
async function generateFeedback(donation: Donation) {
  if (donation.feedbackStatus === 'notified') return;

  // 查找该捐款关联的帮扶记录（通过project_id + 时间窗口关联）
  const aidRecords = await db.beneficiaryAidRecords.findMany({
    where: {
      projectId: donation.projectId,
      aidDate: { gte: donation.paidAt },
      // 关联逻辑：捐款时间之后、同项目的帮扶
    },
    include: { beneficiary: true },
  });

  if (aidRecords.length === 0) return;  // 还没执行帮扶，暂无反馈

  const region = aidRecords[0].beneficiary.city || aidRecords[0].beneficiary.province;
  const content = `您的${formatAmount(donation.amount)}元捐款已用于帮助${region}${aidRecords.length}名受助对象`;

  await db.donations.update({
    where: { id: donation.id },
    data: {
      feedbackStatus: 'notified',
      feedbackContent: content,
      feedbackSentAt: new Date(),
    },
  });

  // 发送订阅消息
  await sendSubscribeMessage(donation.userId, {
    templateId: 'FEEDBACK_TEMPLATE_ID',
    data: {
      thing1: { value: donation.donationNo },
      thing2: { value: content },
      amount3: { value: formatAmount(donation.totalImpact || donation.amount) },
    },
  });
}
```

### 7.4 尊严支付流程 `[可选]`

```
项目设置中开启 dignity_payment_enabled=true, dignity_payment_amount=1(分)

受助人领取帮扶时：
  1. 创建帮扶记录，生成"尊严支付订单"（金额=项目的dignity_payment_amount）
  2. 受助人微信扫码或小程序内支付（1分钱等象征金额）
  3. 支付成功后，帮扶记录确认生效
  4. 帮扶记录中记录 dignity_paid_amount 和 dignity_transaction_id
```

// 数据查询中间件：根据用户角色自动注入数据范围
function applyDataScope(query: any, user: User) {
  if (user.role === 'partner') {
    // 合作机构人员只能看自己机构录入的数据
    query.where = {
      ...query.where,
      partnerOrgId: user.partnerOrgId,
    };
  } else if (user.role === 'recorder') {
    // 内部录入员只能看自己录入的
    query.where = {
      ...query.where,
      recorderId: user.id,
    };
  }
  // admin/manager/super_admin 看全部
  return query;
}
