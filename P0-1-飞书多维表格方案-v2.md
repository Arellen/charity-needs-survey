# P0-1：问卷结果持久化存储 v2

> v1 的问题：一个表格存所有项目，混在一起没法区分。
> v2 改为：**每个项目一张独立工作表（Sheet）**，同一套问卷模板可复用到多个项目。

---

## 一、数据结构设计

```
多维表格：「项目需求存档」
├── 📊 工作表「公益小程序」（当前项目）
├── 📊 工作表「XX助学项目」（未来项目）
├── 📊 工作表「XX医疗项目」（未来项目）
└── 📊  ... 每新增一个项目，新建一张工作表
```

每张工作表的字段按对应项目的问卷题目来。不同项目的问卷题目可能不同（比如公益小程序有"企业配捐"，助学项目可能没有）。

---

## 二、如何区分项目

在问卷页面 URL 上加 `?project=xxx` 参数：

```
公益小程序问卷：
  https://arellen.github.io/charity-needs-survey/survey.html?project=charity

未来助学项目问卷：
  https://arellen.github.io/charity-needs-survey/survey.html?project=edu2026
```

SCF 云函数收到请求后：
1. 读 `project` 参数
2. 查配置表 → 找到对应的飞书 Webhook 群 + 多维表格工作表
3. 发消息到对应群 + 写数据到对应工作表

---

## 三、SCF 配置结构

```python
# 项目配置表（每新增一个项目，加一条）
PROJECTS = {
    "charity": {
        "name": "公益小程序",
        "webhook_url": "https://open.feishu.cn/open-apis/bot/v2/hook/16381057-...",
        "webhook_secret": "RbVRjrUpzmvOI63SL1RQSc",
        "bitable_app_token": "K9Gdxxxxxxxxxxx",
        "table_id": "tblxxxxxxxx",    # 对应「公益小程序」那张工作表
    },
    "edu2026": {
        "name": "2026助学项目",
        "webhook_url": "https://open.feishu.cn/open-apis/bot/v2/hook/xxxxx",
        "webhook_secret": "xxxxxx",
        "bitable_app_token": "K9Gdxxxxxxxxxxx",   # 同一个多维表格
        "table_id": "tblyyyyyyyy",                 # 不同工作表
    },
}
```

新增一个项目：在 `PROJECTS` 里加一条配置 + 在飞书里新建一张工作表 → 重新部署 SCF。

## 四、复用流程：新项目怎么搭

假设你现在对接第二个机构/项目：

| 步骤 | 操作 | 耗时 |
|---|---|---|
| 1 | 复制 `survey.html` → 改名，修改问卷题目 | 20 分钟 |
| 2 | 飞书里建新工作表（复制旧表的字段结构） | 2 分钟 |
| 3 | 飞书里建新群 + 新机器人 | 2 分钟 |
| 4 | 在 SCF 的 `PROJECTS` 里加一条配置 | 1 分钟 |
| 5 | 部署 SCF | 1 分钟 |
| 6 | 把新问卷 URL 发给新机构 | 完成 |

## 五、方案里还需要补充的

### 5.1 问卷页本身需要展示项目名

当前 survey.html 没有项目名标识。加上：
- 页面标题旁显示项目名（从 URL 参数读取）
- 提交后的确认页也显示项目名
- 飞书消息开头标注项目名

### 5.2 字段不需要 30 个那么多

当前方案里每个项目一张工作表，字段跟问卷题目一一对应就够了。但有些字段是**所有项目共有的**，可以在总览表里汇总：

```
多维表格：「项目需求存档」
├── 📊 工作表「总览」（所有项目的摘要，方便全局查看）
│   字段：提交时间 | 项目名 | 机构名称 | 填表人 | 联系方式 | 核心需求摘要 | 期望上线 | 预算 | 版本标记
│
├── 📊 工作表「公益小程序」（完整字段，和问卷一一对应）
├── 📊 工作表「2026助学项目」
└── ...
```

总览表只存关键字段，方便你一眼扫完所有项目；详细表存完整问卷数据，需要细节时点进去看。

## 六、总结：v1 vs v2

| 维度 | v1 | v2 |
|---|---|---|
| 项目区分 | ❌ 混在一起 | ✅ 每个项目一张工作表 |
| 多项目复用 | ❌ 不支持 | ✅ URL参数 + 配置表 |
| 全局视图 | ❌ 没有 | ✅「总览」表跨项目查看 |
| 字段灵活性 | ❌ 所有项目同一套字段 | ✅ 每张表按需设计 |
| 新增项目成本 | - | 复制问卷+建表+加配置 ≈ 30 分钟 |
